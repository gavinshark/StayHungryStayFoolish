## 📁 文档组织规范

### 规则

1. **所有文档文件（除 README.md）都放在 doc/ 目录下**
2. **根目录只保留 README.md**
3. **build/ 目录可以有自己的 README.md**
4. **tests/ 目录可以有自己的 EXAMPLES.md**

### 文档命名规范

- 使用大写字母和下划线：`PROJECT_STATUS.md`
- 描述性名称：`LOG_CONFIGURATION.md`
- 避免使用空格

### 文档结构

每个文档应包含：

1. **标题** - 清晰的文档标题
2. **概述** - 简短的说明
3. **详细内容** - 分节组织
4. **示例** - 实际使用示例（如适用）
5. **相关链接** - 链接到其他相关文档
6. **更新日期** - 最后更新时间

### 示例模板

```markdown
# 文档标题

## 概述

简短的说明...

## 详细内容

### 小节 1

内容...

### 小节 2

内容...

## 示例

```bash
示例代码...
```

## 相关文档

- [其他文档](OTHER_DOC.md)

---

**最后更新**: YYYY-MM-DD
```

## 🔍 查找文档

### 按主题查找

- **构建相关**: BUILD_*.md, COMPILE_*.md
- **配置相关**: LOG_*.md, *_CONFIGURATION.md
- **项目信息**: PROJECT_*.md, STRUCTURE.md

### 快速访问

```bash
# 查看所有文档
ls doc/

# 搜索文档内容
grep -r "关键词" doc/

# 查看文档列表
cat doc/README.md
```

## 📝 添加新文档

当添加新文档时：

1. 在 `doc/` 目录创建文件
2. 使用描述性的大写文件名
3. 在本 README.md 中添加条目
4. 在主 README.md 中添加链接（如需要）

### 示例

```bash
# 创建新文档
vim doc/NEW_FEATURE.md

# 更新文档索引
vim doc/README.md

# 更新主 README（如需要）
vim README.md
```
